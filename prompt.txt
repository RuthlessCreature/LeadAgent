# 一、输入层（Agent 输入规范）

这个模块处理用户定义产品/客户信息的**结构化输入**：

### 1. 产品描述解析

**目标**：从自然语言中提取可比较结构化的字段
**输入**：用户自然语言产品文本
**输出结构**：

```
{
  product_name,
  industry_tags,
  feature_tags,
  use_cases,
  target_roles,
  price_range,
  exclude_tags
}
```

**逻辑要求**：

* LLM 提取实体并标准化为数据库统一标签
* 多语言输入支持（中英→统一编码）
* 含不相关词剔除（stop-word）

---

### 2. 理想客户画像（ICP）定义器

**目标**：建立 Ideal Customer Profile
**字段**：

| 字段               | 类型        | 说明       |
| ---------------- | --------- | -------- |
| geography        | list      | 国家/地区    |
| company_size     | range/int | 员工数区间    |
| industry         | list      | 行业       |
| role_titles      | list      | 决策/影响人岗位 |
| revenue_range    | range     | 营收区间     |
| technology_stack | list      | 与产品匹配的技术 |

**输出**：标准化 ICP JSON
**用途**：驱动后续 search & scoring

---

# 二、搜索与执行层（Agent Task Spec）

这个层是你 agent 的 “执行引擎”。每个模块都要明确输入/过程/输出。

---

### 3. 多平台目标搜索

支持平台：

* LinkedIn People/Company Search
* Facebook / Groups / Marketplace
* TikTok Creator/Business Pages
* YouTube Channels
* Google Search / B2B Sites
* B2B 数据库 (ZoomInfo/Apollo/Crunchbase) ([BuyerSprint Hub][1])

**输入**：search query + ICP
**过程**：

1. 生成平台专用查询词
2. 执行自动浏览器/接口调用
3. 提取 DOM/raw JSON

**输出**：候选公司/联系人列表

---

### 4. 爬取与反爬配置

**必需组件**：

* 代理池与指纹管理
* 分布式调度
* CAPTCHA 处理集成
* 请求速率控制

**输出**：结构化 HTML/text

---

### 5. 初级抽取器（DOM/Text → JSON）

针对不同平台分别解析：

| 平台              | 抽取字段                  |
| --------------- | --------------------- |
| LinkedIn        | 公司名 / 联系人 / 角色 / 公司规模 |
| Facebook        | 页面名 / Group成员 / 联系邮箱  |
| TikTok          | 账号名 / bio / 地区标签      |
| Google B2B page | 公司 / 电话 / 邮件 / 产品     |

输出统一格式：

```
{
  company,
  contact_name,
  role,
  platform,
  raw_text_snippet
}
```

# 三、评分 & 过滤引擎（核心商业价值）

这个是你商业化卖点：**给线索打分**。不是简单筛选。

---

### 6. 行业匹配评分器

**输入**：公司行业 + 产品行业
**规则**：

* 精确匹配 + 模糊匹配
* 使用 vector similarity
* 动态可配置权重

**输出**：industry_score (0-100)

---

### 7. 采购意图识别

判断一个 lead 是否真正有购买需求：

**信号列表**：

* 招聘采购/商务相关岗位
* 发布“Looking for suppliers”帖子
* 产品询价
* 社交媒体表达采购意图

**输出**：intent_score (0-100)
**规则**：结合 NLP classification + keyword triggers

---

### 8. 联系可达性评估

**评估是否可以联系这个线索**：

| 来源            | 可能性评分维度    |
| ------------- | ---------- |
| Email         | 格式合法 / 验证率 |
| LinkedIn DM   | 一级连接优先     |
| 社交 Profile DM | 活跃度        |

输出：contact_score

---

### 9. 总体 Lead-Score 计算器

**公式**（可调整参数）：

```
total_score = w1*industry_score
            + w2*intent_score
            + w3*contact_score
```

输出：sorted list

# 四、细化功能模块（从前端到后台）

---

## 10. 客户情报卡（Lead Card）

每个‘lead’呈现为完整情报卡，字段必须齐全：

```
{
  company,
  contacts: [{name, role, email, linkedin}],
  product_fit_summary,
  scores: {industry, intent, contact, overall},
  snippets: [text from web],
  source_platform
}
```

**UI/存储**：可导出 PDF/CSV

---

## 11. 多平台重复/去重系统

规则：

* 域名标准化
* 公司名 fuzzy match (Levenshtein)
* contact email 唯一性
  输出：清洁客户池

---

## 12. 多轮搜索策略器（Agent 大脑）

策略能力：

* 根据当前得分分布决定是否继续搜索
* 自动调整 query 优化
* 自动选择下一个 platform 重试

**输出**：新的任务 queue

---

## 13. 任务队列与状态跟踪

字段：

```
task_id,
platform,
query,
status,
results_count,
last_run
```

---

## 14. 意图预测与分类

LLM 分类 API 负责：

* 行业分类
* 采购信号标记
* 角色职位分类
